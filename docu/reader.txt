ВВЕДЕНИЕ
--------

   SWAG (SourceWare Archival Group) - это коллекция исходных кодов и
программных примеров для языка программирования PASCAL, выпускаемая Гэйлом
Дэвисом (Gayle Davis, GDSoft, gdavis@gdsoft.com, http://www.gdsoft.com).  
Этот проект до сих пор является одним из самых популярных в среде 
программирующих на Паскале.

   Russian SWAG (RSWAG) - это русская редакция популярной англоязычной версии.
Коллекция Russian SWAG пополняется материалами из различных сетевых
телеконференций и лично многими программистами exUSSR, пожелавшими внести свой
посильный вклад в популяризацию одного из самых лучших языков программирования.

   Очередные выпуски RSWAG выходят в свет по мере накопления достаточного
количества материалов (обычно 1 раз в квартал) и доступны для пользователей на
многих BBS, FTP, WWW и Мэйл-серверах.
Всю коллекцию RSWAG (и SWAG тоже) можно найти на нашей страничке:
http://rswag.tvcom.ru


НЕМНОГО ИСТОРИИ...
------------------

   До 1994г Russian SWAG был полностью совместим со своим англоязычным
собратом. В русифицированную версию READER.EXE были внесены лишь небольшие
изменения, касающиеся только русификации проекта.  Все остальное (в т.ч.
интерфейс, формат хранения данных, горячие клавиши, режимы и алгоритмы...) было
сохранено, так сказать, в первозданном виде.

   Однако, в связи с нежеланием авторов оригинального SWAG поддержать мою идею
о интернационализации их проекта, дальнейшее развитие российского проекта
оказалось весьма проблематичным и было даже приостановлено на неопределенный
срок...

   Своим возрождением проект Russian SWAG обязан множеству писем от
заинтересованных программистов со всего exUSSR с пожеланием непременно
продолжить выпуски RSWAG.

   Таким образом, благодаря Вам, Russian SWAG снова пополняется и публикуется!
Но, поскольку англоязычный SWAG с 1997г перестал выпускаться, и поддержки
никакой от SWAG-овцев не было, пришлось полностью переработать и формат
представления данных, и собственно программу просмотра.

   Итак, что изменилось с выпуска 94г:

1) С целью универсализации доступа из разных программных сред (Паскаль,
   Delphi, FoxPro, Clipper, Basic, Access...) был выбран формат DBF,
   а точнее - FoxPro с Мемо-полями в формате FPT и компактными
   индексами типа CDX.
   Почему был сделан такой выбор? Потому, что теперь любой программист
   сможет написать свою собственную оболочку для SWAG (хоть на Паскале,
   хоть на Clipper/FoxPro/Basic/Access....).

2) Для сокращения занимаемого базами данных дискового пространства
   было решено хранить тесты сообщений в Мемо-полях в сжатом виде.
   Для сжатия информации использовалась библиотека "PkWare Data
   Compression Library v1.02", имеющая интерфейс для Pascal,
   Clipper, Basic, MS C++, Lattice C++, Assembler. Выбрана она была
   именно из-за своей многоплатформенности.

3) Была полностью переписана собственно оболочка READER.EXE. Сначала
   было сделано 2 совместимых версии: одна на Паскале, другая на
   Клиппере, но впоследствии осталась только Паскальная версия,
   поскольку ее размер оказался в 6-8 раз меньше. Да и собственно
   сам SWAG создавался для Паскальных исходников :)

   Внешне новая оболочка стала похожа на хорошо известный любому
   фидошнику GoldEd by Odinn Sorensen. Сохранены, насколько возможно,
   практически все режимы и горячие клавиши. Так что, использование
   RSWAG для фидошников не составит никакого труда (см. HELP по F1).

   Тем, кто не знаком с этой знаменитой программой ("Голый Дед" :-),
   новый READER тоже не доставит особых хлопот, поскольку интерфейс
   у этой программы вполне понятный, почти интуитивный...

4) По результатам бета-тестирования в выпуске RSWAG-96 был несколько
   изменен формат баз данных, а именно добавлены поля для дальнейшего
   хранения "приаттачченных" двоичных файлов (с целью отказа от
   использования UuEncoded файлов, поскольку они на 30-40% увеличивают
   объем хранимой информации по сравнению с оригинальным объемом).

5) В выпуске за 1997 год наконец-то реализован File-Attach (подключение
   двоичных файлов без их преобразования в текст с помощью UUEncode).
   Теперь каждая статья может иметь текст описания размером до 64К
   плюс бинарный файл размером до 2х Гигабайт.

НАЧАЛО РАБОТЫ
-------------

   Программа READER.EXE (далее READER) предназначена для чтения, извлечения и
печати статей из архива RSWAG. Весь архив RSWAG разбит на тематические пакеты
(файлы), каждый из которых содержит несколько статей по данной тематике.

   Файлы проекта RSWAG желательно разместить вместе в одном директории на Вашем
диске. Рекомендуем создать для этого директорий с именем \RSWAG.  Для этого Вы
можете использовать команду ДОС:

                  mkdir c:\rswag

   Распакуйте дистрибутивный архив ALLRSWAG.ZIP в директории \RSWAG c помощью
следующей команды:

                  pkunzip -d allrswag.zip

   После распаковки Вы получите следующий набор файлов:

                \RSWAG          - основной каталог Russiаn SWAG
                  reader.exe    - программа чтения
                  reader.ini    - установочные параметры RSWAG
                  file_id.diz   - краткое назначение RSWAG
                \RSWAG\DATABASE - каталог для тематических баз
                  *.dbf         - файлы заголовков сообщений
                  *.fpt         - файлы с текстами сообщений
                  *.cdx         - вспомогательные индексные файлы
                \RSWAG\DOCS     - документация к Russian SWAG
                  file_id.diz   - краткое назначение RSWAG
                  reader.txt    - Описание работы с READER.EXE
                  bbs.txt       - Памятка владельцам BBS и FTP/WWW серверов
                  rswag.txt     - Описание проекта Russian SWAG
                  howtouse.txt  - Инструкция по распаковке архива
                \RSWAG\EXTRACT  - каталог для выдирания файлов


   _П_Р_И_М_Е_Ч_А_Н_И_Е_

             Если у Вас в каталоге DATABASE\ сохранились индексные
файлы от старой версии: *.ID1, *.ID2, *.CDX - просто удалите их!
Программа Reader просто пересоздаст их заново.

УДАЛЕНИЕ ТЕМАТИЧЕСКИХ ФАЙЛОВ
----------------------------

   Вы имеете возможность удалить из архива RSWAG целиком тематические файлы,
которые Вас не интересуют.  Для этого используется клавиша Delete.
Будьте осторожней с этой возможностью - восстановить удаленное не удастся :-(

     ┌── Delete ───╖
     │ Marked Areas║
     │ Current Area║
     │ Quit  / ESC ║
     ╘═════════════╝

ДОБАВЛЕНИЕ НОВЫХ ТЕМАТИЧЕСКИХ ФАЙЛОВ
------------------------------------

   Вы имеете возможность добавить в архив RSWAG новые тематические файлы по
своему собственному усмотрению.  Для этого используется клавиша Insert.  При ее
нажатии без лишних разговоров добавляется новая ПУСТАЯ строка в списке тем.  Ее
нужно будет лишь подкорректировать (ввести идентификатор и описание).
Собственно тематический файл сразу не будет создан, но при первой же попытке
войти в эту тему (клавиша Enter) Вам будет предложено создать новый файл.
Если Вы оставите имя темы пустым, то открыть такую базу, разумеется,
не удастся :)


КОРРЕКТИРОВКА ТЕМАТИЧЕСКИХ ФАЙЛОВ
---------------------------------

   Изменить идентификатор (AreaID) и/или описание (Description) тематического
файла можно по команде Alt-C (Change). Установите указатель на нужную строку и
нажмите Alt-C. Внесите необходимые исправления. Если Вы измените идентификатор
уже существующего файла, то он, соответственно, будет переименован.


  ┌─ Change this Area? ────╖
  │░░░░░░Yes Please.░░░░░░░║
  │          NO!           ║
  ╘════════════════════════╝

ПОМЕТКА СТАТЕЙ
--------------

   Интересующие Вас статьи можно пометить Пробелом.
Помеченные статьи могут затем быть подвергнуты групповой обработке:
удалению (Del), копированию или перемещению (Alt-M), выводу в файл (Alt-W).


ПОИСК СТАТЕЙ
--------------------

   Когда перед Вами список статей, для быстрого поиска нужной статьи Вы можете
использовать клавишу Alt-Z и затем ввести искомую строку.

       ┌─Enter the string for Quick search:──────────────────────────╖
       │░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░║
       ╘═════════════════════════════════════════════════════════════╝

   Поиск в этом режиме осуществляется только по заголовкам статей.  Если статья
будет найдена, то курсор встанет на нужную Вам статью. 


ПОИСК ТЕКСТА В СТАТЬЯХ
----------------------

   Для поиска текста в статьях используйте кнопочку Alt-F (Find). Затем введите
искомую подстроку.

       ┌─Enter the search string:────────────────────────────────────╖
       │░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░║
       ╘═════════════════════════════════════════════════════════════╝

   Если искомый текст будет найден, то будет показана найденная статья.  А для
повторного поиска - как обычно в Борландовском Паскале - Ctrl-L.




СТАТЬИ, ПОМЕЧЕННЫЕ ГАЛОЧКОЙ
---------------------------

   Если при просмотре списка статей Вы увидели возле некоторых галочки (символ
#251) - не пугайтесь :-) ! Это просто так помечаются НОВЫЕ статьи, которые были
добавлены при последнем UPDATE-е. READER помечает их автоматически. Просто,
чтоб Вы в курсе были.

 Ctrl-Home, <   Самое первое сообщение в группе
 Ctrl-End,  >   Самое последнее сообщение в группе



ВЫВОД СТАТЕЙ НА ДИСК ИЛИ ПРИНТЕР
--------------------------------

   Статьи (текущая или помеченные) могут быть выведены  в отдельный ASCII 
файл нажатием клавиши Alt-W (Write). 

	┌─── Write ───┐
	│ Marked Msgs │
	│ Current Msg │
	│ Quit  / ESC │
	└─────────────┘

Затем можно выбрать, куда именно вывести текст:

	┌─── Write To: ───┐
	│ Disk File       │
	│ Print device    │
	│ Clipboard       │
	│ Use Header: YES │
	│ Quit   / ESC    │
	└─────────────────┘

При этом можно уточнить, надо ли писать заголовки статей (Use Header).

Затем нужно будет лишь указать имя файла, в который будет произведена
запись. Если такой файл уже существует, Вам будет предложено добавить новый
текст в конец существующего файла (APPEND), или перезаписать его (OVERWRITE).

	┌───── File Exists! ──────┐
	│ Append to end of file   │
	│ Overwrite the file      │
	│ Cancel  / ESC           │
	└─────────────────────────┘

ЧТО ДЕЛАТЬ С "АБРАКАДАБРОЙ" В СТАТЬЯХ
-------------------------------------

   Многие статьи в коллекции RSWAG содержат куски, которые выглядят в виде
упорядоченной "абракадабры".  Не пугайтесь - это закодированные в текстовую
форму двоичные файлы. Обычно они выглядят примерно так:


section 1 of uuencode 4.13 of file READER.INI    by R.E.M.

begin 644 READER.INI
M0T],3U(]3$E'2%1'4D%9#0HT,R\U,#U.3PT*4T]25#U.54U"15(-"E1%6%1%.
.......
14PT*4TA/5U-40513/4Y/#0I/D
``
end
sum -r/size 48328/244 section (from "begin" to "end")
sum -r/size 48361/152 entire input file


"Ну, и что мне со всем этим делать?" - спросите Вы. А вот что:

1) Старый "дедовский способ":

   Аккуратно вырежьте такие куски каждый в отдельный файл (желательно с
расширением). И напустите на них специальный раскодировщик - UUDECODE.EXE,
который вы найдете в комплекте поставки RSWAG.

              UUDECODE имя_вырезанного_файла

И Вы получите раскодированный двоичный файл, имя которого указано в
строчке

begin 644 ИМЯ_ФАЙЛА

Кстати, Вы можете использовать для этой цели и любую другую
uuencode/uudecode - совместимую программу.

2) Но можно сделать проще! (с Рондо! ;) (встроенное декодирование):

   Просто нажмите Alt-U (UuDecode), и все такие файлы будут автоматически
декодированы и записаны в директорий  EXTRACT. Можно воспользоваться еще и
командой Extract (Alt-E). В этом случае из сообщения будут выдраны вообще 
ВСЕ входящие в него файлы.



ЧТО ДЕЛАТЬ С МНОГОФАЙЛОВЫМИ СТАТЬЯМИ
------------------------------------

   Некоторые статьи в коллекции RSWAG содержат в себе не просто текст, а целый
проект, состоящий из нескольких файлов. Перед каждым файлом вначале ставится
специальная строка-разделитель с именем файла:


{>■■■■■■■■■■ Cut here. FileName= UN_PACK.PAS ■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■}

   Чтобы получить из такой статьи многофайловый проект, нужно сделать
следующее:

1) Старый "дедовский способ":

   Вывести статью в файл (Alt-W), текстовым редактором вырезать все входящие в
статью файлы, задавая указанные в разделителях имена.  Если некоторые файлы
будут в формате UuEncoded (расширение .UUE), то нужно декодировать их с помощью
UUDECODE.EXE. (уфффф!)

2) Более простой способ (Команда Extract):

   Просто на интересующей Вас статье нажмите Alt-Е (Extract), уточните имя
каталога:

       ┌─Extract to the Directory:───────────────────────────────────╖
       │EXTRACT\░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░║
       ╘═════════════════════════════════════════════════════════════╝


...и все файлы, содержащиеся в статье, будут автоматически вырезаны и помещены
в директорий EXTRACT. Кроме того все UuEncoded файлы также будут декодированы в
исходный двоичный вид.

КОРРЕКТИРОВКА СТАТЕЙ
--------------------

   Для корректировки (внесения изменений) текущей статьи используется клавиша
Alt-C (Change). После нажатия Alt-C от Вас потребуется подтверждение
необходимости редактирования:

  ┌─ Change this record? ──╖
  │      Yes Please.       ║
  │          NO!           ║
  ╘════════════════════════╝

   В случае подтверждения Вы сможете исправить сначала заголовок статьи, а
затем и собственно текст статьи. При редактировании заголовка статьи
можно использовать клавиши перемещения по полям:

Вперед - Enter, TAB, Down (Стрелка вниз)
Назад  - Shift-TAB,  Up   (Стрелка вверх)

А для перемещения внутри редактируемого поля - Right, Left, BackSpace.

ВСТАВКА ТЕКСТОВОГО ФАЙЛА В ТЕКСТ СТАТЬИ
---------------------------------------

   Для вставки в статью с текущей позиции содержимого какого-либо файла
нажмите Alt-I (Insert), и выберите, в каком виде его нужно вставлять.
Двоичный файл в текст статьи просто так вставить нельзя (ведь вставляется-то
не текст!). Для этого нужно сначала преобразовать двоичный файл в текстовую
форму (UuEncode), и лишь затем вставить в статью полученный код.
Однако, Вам не придется проделывать это вручную. Программа READER сделает это
за Вас. Для этого выберите в меню пункт Binary as UUE.

	┌──── Import ────┐
	│ File as Text   │
	│ Binary as UUE  │
	│ Binary Attach  │
	│ Nothing / Quit │
	└────────────────┘

Затем выберите из списка имя вставляемого файла. 

В строке перед втавляемым файлом автоматически запишется строка-разделитель:

{■■■■■■■■■■ Cut here. FileName= UN_PACK.PAS ■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■>}


Если вставляемый файл является двоичным (Выполняемый модуль или архив),
то следует воспользоваться:
 - пунктом меню "Binary as UUE" (преобразование в текстовый вид 
   и включение непосредственно в текст статьи), если файл имеет размер
   не более 30-40Кb
 - пунктом "Binary Attach" для подключения этого файла без каких-либо 
   преобразований. В этом случае Файл помещается не в текст статьи,
   а отдельно в мемо-файл *.FPT. При этом в заголовке статьи появится
   указатель:
              Attachment:[FILENAME.ZIP]

ФАЙЛ READER.INI
---------------

   Основные парпаметры READERа хранятся в файле READER.INI (текстовом) в том же
директории, где он сам расположен. При желании, Вы можете этот файл
подкорректировать по своему усмотрению. Формат строк в файле не простой, а
очень простой:

       Parameter = VALUE      ; Comment

В файле READER.INI Вы увидите что-то типа :


;------- RSWAG Configure Parameters ----------------------------------

[ Work Pathes ]
BaseDir    = DATABASE\  ; Path to DataBases
                        ; путь к базам данных
ExtractDir = EXTRACT\   ; Default Directory for Extraction
                        ; путь для помещения проектов
ImportDir  = IMPORT\    ; Default Directory for Import
                        ; путь для импортируемых файлов
ListDir    =            ; Default Directory for HTML Lists
                        ; путь для списков статей в HTML формате

[ Other Parameters ]

;AreaOrder  = 0         ; Not Ordered Area List (default)
;AreaOrder  = 1         ; Order Area List by DESCRIPTION
AreaOrder  = 2          ; Order Area List by AREAID
;MsgOrder   = 0         ; Not Ordered Msg List (default)
;MsgOrder   = 1         ; Order Msg List by FROM
MsgOrder   = 2          ; Order Msg List by SUBJECT

LastUpdate =            ; Last updated by PATCHNAME  

В общем, идея понятна. Редактируется любым текстовым редактором.

повседневной практике программирования на Паскале.
